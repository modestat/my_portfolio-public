Assignment 1: BEM Methodology

Link to the Assigment 1

In this first assignment, I utilized the BEM (Block Element Modifier) methodology to enhance my understanding of structured CSS naming conventions. The design system of the finn.no webpage played a crucial role in facilitating this process. It provided clear guidelines for elements such as primary buttons, secondary buttons, and a simple footer with links and inputs, making it easier to define blocks, elements, and modifiers.

Application in the Assigment

The use of a design system helped me consistently apply BEM naming conventions throughout the assignment. Here are some specific examples from my code:

  • Blocks: The main structural components such as .main, .footer, and .form are clearly defined as blocks. They served as foundational elements of the page layout.

  • Elements: Within these blocks, elements are defined to structure the content further. For instance, .main__section--left, .form__input-container1, and .footer__link-item are elements that build upon their respective blocks, ensuring a logical hierarchy.

  • Modifiers: Modifiers like .main__section--left are used to create variations of blocks or elements, adding specificity and flexibility to the design without disrupting the overall structure. Overall, the integration of the design system with BEM allowed for a more organized and maintainable codebase. By having predefined styles for primary and secondary buttons, and a structured layout for the footer, I was able to streamline the development process. This approach not only improved the clarity and readability of my code but also made it easier to manage and update as needed.

Reflection

Using BEM helped in creating a more organized and maintainable codebase. The predefined styles for primary and secondary buttons, along with a structured layout for the footer, streamlined the development process. This approach improved the clarity and readability of my code, making it easier to manage and update.

Areas for Improvement

One area I could improve is giving class names to paragraphs to make the CSS code more efficient. This would allow for better targeting of specific paragraphs within a block-element structure.
